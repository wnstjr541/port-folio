  • 사용자가 웹사이트로 정보를 보낼수 있는 요소 ex) 회원가입

    태그 정리

  • action : 폼을 전송할 서버쪽 스크립트 파일 지정
  • name : 폼을 식별하기 위한 이름을 지정
  • accept-charset : 폼 전송에 사용할 문자 인코딩을 지정 (uft-8)
  • target : action에서 지정한 스크립트 파일을 현재창이 아닌 다른위치에 열도록 지정합니다
  • method : 폼을 서버에 전송할 http메소드를 정합니다(get 또는 post)

    HTTP Method란

  • 클라이언트와 서버 사이에 이루어지는 요청(Request)과 응답(Response) 데이터를 전송하는 방식. (서버에 요청을 보내는 방법)
  • ge: 폼 데이터를 url 끝에 붙여서 눈에 보이게 보낸다
  • post : 내부적으로 보이지 않게 보낸다.
  • 보안이 필요하지 않으면서 지정된 리소스에서 자원을 읽는 경우 get 그 외 보안이 필요한 경우는 post

    input 기본형태

  • p
    label for="userID" /label
    input type="text" name="userID" id="userID" placeholder="이름을 입력해주세요" required="required" autofocus="auto"/> /p
  • label 태그는 사용자 인터페이스(UI) 요소의 라벨(label)을 정의할 때 사용합니다.label 요소는 for 속성을 사용하여 다른 요소와 결합하고 label 요소를 결합하고자 하는 요소 내부에 위치시키면 for 속성을 사용하지 않더라도 해당 요소와 결합시킬 수 있습니다.
    label 요소를 사용할 수 있는 요소는 다음과 같습니다.
    - button, input, meter, output, progress, select, textarea
  • name : 서버로 전송될 라디오 및 체크박스의 이름. key 값에 해당함.라디오 버튼은 동일한 그룹끼리 같은 name 값을 지정함.
  • value : 서버로 넘어갈 값을 지정.
  • checked : 초깃값으로 기본 선택될 항목을 설정.
  • required : 필수 작성 사항
  • disabled 속성은 해당 input 요소가 비활성화됨을 명시, 폼 데이터가 제출될 때도 disabled 속성이 명시된 input 요소의 데이터는 제출되지 않습니다.
  • readonly 속성은 input 요소의 입력 필드가 읽기 전용임을 명시
  • radio : 둥근 버튼 생성. 단일 선택 가능
  • checkbox : 체크 박스(사각형) 생성. 다중 선택 가능
  • button 4가지 : image,submit,reset,button
  • fieldset 태그는 form 요소에서 연관된 요소들을 하나의 그룹으로 묶을 때 사용합니다.
  • legend 태그는 fieldset 요소의 캡션(caption)을 정의할 때 사용합니다.select 태그의 form 속성은 해당 드롭다운 리스트가 포함될 하나 이상의 form 요소를 명시합니다.
  • ex)select name="order" form="myForm"
